Features

Everything under the hood

Real blockchain mechanics, explained in code you can actually read.

⛏

Proof-of-Work Mining

Brute-force nonces until the SHA-256 hash meets the difficulty target. Watch it happen in real time with configurable difficulty.

🔑

Ed25519 Signatures

Real cryptographic signatures on every transaction. Private keys sign, public keys verify. Not a simulation.

💰

Native Coin (ANV)

Mining rewards create new coins. Send between wallets. Track balances. All sandbox — zero real value.

🔗

Tamper-Proof Chain

Each block stores the previous hash. Change one byte and every subsequent block breaks. Immutability by design.

🌐

Multi-Node Network

Spin up 2, 3, or more local nodes. They broadcast blocks and reach consensus via longest-valid-chain rule.

⚙

One Config File

Coin name, ticker, difficulty, block reward, block size — all in config.js. Fork it and make your own chain.

How It Works

From zero to blockchain

Five steps. Each one builds on the last.

1. Create a Wallet

Generate an Ed25519 key pair. Your public key is your address. Your private key signs every transaction you make.

anvil > create-wallet

2. Mine a Block

Pick up pending transactions, add a coinbase reward, and increment a nonce until the hash starts with enough zeros.

anvil > mine

3. Send Coins

Sign a transaction with your private key. The chain checks your balance and signature before accepting it.

anvil > send <address> 25

4. The Chain Grows

Each new block links to the previous one through its hash. Tamper with block #3 and blocks #4, #5, #6... all become invalid.

anvil > chain

5. Network Consensus

Run multiple nodes. They share chains over HTTP. The longest valid chain wins — Nakamoto consensus.

PORT=3002 PEERS=http://localhost:3001 npm run node
