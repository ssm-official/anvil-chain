What's Inside

Everything you need to understand blockchain fundamentals. Nothing you don't.

⛏

Proof-of-Work Mining

Increment a nonce until the hash has enough leading zeros. Configurable difficulty so you can watch it happen in real time.

🔑

Ed25519 Signatures

Every transaction is signed with a real Ed25519 private key and verified with the public key. Actual cryptography, not a placeholder.

💰

Native Coin (ANV)

Miners earn block rewards. Send coins between wallets. Track balances. All in a sandbox with zero real-world value.

🔗

Tamper-Proof Chain

Each block stores the hash of the previous one. Change a single byte and the chain breaks — visibly and verifiably.

🌐

Multi-Node Network

Run 2, 3, or more nodes on localhost. They broadcast blocks, validate chains, and reach consensus automatically.

⚙

One Config File

Change the coin name, ticker, difficulty, block reward, and block size from a single file. Fork it and make it yours.

How It Works

Five steps from nothing to a working blockchain.

1. Create a Wallet

Generate an Ed25519 key pair. Your public key becomes your address. Your private key stays secret and signs transactions.

anvil > create-wallet

2. Mine Your First Block

The miner picks up pending transactions, adds a coinbase reward, and brute-forces a nonce until the hash meets the difficulty target.

anvil > mine

3. Send Coins

Create a transaction, sign it with your private key, and submit it to the pending pool. The chain checks your balance to prevent double-spending.

anvil > send <address> 25

4. Chain Grows

Each new block links to the previous one via its hash. A tampered block would change every hash after it — the chain becomes self-evident.

anvil > chain

5. Network Consensus

Multiple nodes share their chains over HTTP. The longest valid chain wins. This is the same principle Bitcoin uses — Nakamoto consensus.

PORT=3002 PEERS=http://localhost:3001 npm run node
